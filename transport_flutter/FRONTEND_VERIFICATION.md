# Flutter 前端驗證指南

## 概述

Flutter 前端已經更新以支援新的 CSV 資料來源。主要變更包括：

1. **BusApiService** - 新增 `searchBusRoutes()` 方法
2. **BusListProvider** - 增強搜尋功能，支援搜尋模式
3. **BusListScreen** - 改善 UI，顯示更多路線資訊

## API 相容性

現有 API 呼叫已與新後端相容：

| 方法 | 端點 | 狀態 |
|-----|------|------|
| `fetchBusRoutes()` | `GET /api/bus/routes` | ✅ 已相容 |
| `fetchBusRouteData()` | `GET /api/bus/{route}` | ✅ 已相容 |
| `searchBusRoutes()` | `GET /api/bus/routes/search` | 🆕 新增 |

## 測試步驟

### 1. 啟動後端伺服器

```bash
cd backend
python main.py
```

確認伺服器運行在 `http://localhost:8000`

### 2. 啟動 Flutter 應用

```bash
cd transport_flutter
flutter run
```

### 3. 測試功能

#### 測試 1：路線列表
- 開啟應用，進入「公車路線」頁面
- 應該顯示路線列表（使用 CSV 資料）
- 確認可以看到 626 條路線

#### 測試 2：搜尋功能
- 在搜尋欄輸入 "935"
- 應該顯示符合的路線
- 確認搜尋結果包含路線名稱、起迄站、業者

#### 測試 3：路線詳情
- 點擊任一路線
- 應該顯示路線詳情頁面
- 確認可以看到站牌列表與到站時間

#### 測試 4：方向切換
- 在路線詳情頁面切換去程/返程
- 確認站牌列表正確更新

## 常見問題

### Q1: 搜尋沒有結果？
- 確認後端伺服器運行中
- 檢查網路連接
- 查看後端日誌確認 CSV 資料已載入

### Q2: 站牌資料顯示不正確？
- 確認後端 CSV 檔案已下載
- 檢查 `backend/data/ntpc_bus/` 目錄

### Q3: 到站時間不更新？
- 到站時間來自 CSV 資料（每分鐘更新）
- 下拉重新整理可獲取最新資料

## 效能考量

- CSV 資料在後端啟動時載入
- 路線列表載入時間：< 1 秒
- 搜尋回應時間：< 100ms
- 建議每 30 秒重新整理一次路線詳情

## 錯誤處理

應用已實作以下錯誤處理：

1. **網路錯誤** - 顯示錯誤訊息與重試按鈕
2. **無資料** - 顯示「沒有找到路線」提示
3. **API 失敗** - 自動回退到舊版資料
